# Contribution guide

We are glad you're interested in contributing to Aletheia! To ensure a smooth collaboration, please follow the
guidelines below.

## Contributions we welcome

We welcome contributions in the following areas:

- Bug reports and feature requests.
- Code contributions (new features, bug fixes, improvements).
- Documentation improvements.
- Code reviews and testing.
- Examples and tutorials.

## Code contribution

Feel free to pick an open issue or propose a new one for discussion. When submitting code contributions, please make
your PR follow these guidelines:

### PR guidelines

- The title should follow [conventional commit style](https://github.com/openkairos/documentation/blob/main/docs/technical/conventional-commits.md) (e.g.,
  `feat(shipping): send email to customer on shipment`).
- Fill out the PR template provided.
- Ensure all CI checks pass before requesting a review.
- Write a clear description of the changes made, helping reviewers understand the context.
